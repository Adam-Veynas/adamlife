# AdamLife

## Vision

AdamLife is a personal life management application designed to organize everyday life in a simple, fast, and structured way.

The application is not intended for public release during the first development stage. It is being built primarily for daily personal use. Every feature must solve a real-life problem instead of existing only because it is technically possible.

The primary goal is to reduce daily chaos by keeping all important information in one place.

---

# Core Principles

The application must always remain:

- Fast
- Minimalistic
- Clean
- Easy to understand
- Pleasant to use every day

If a feature makes the application more complicated without providing significant value, it should not be implemented.

---

# Main Modules

The first release includes:

- Dashboard
- Planner
- Finance
- Vehicles
- Settings

Future modules may include:

- Health
- Documents
- Notes
- Habits
- Shopping
- Travel

Future modules must not require changing the existing architecture.

---

# Philosophy

AdamLife should feel like a personal assistant rather than a database.

The interface must never overwhelm the user with information.

Every screen should answer one simple question.

Examples:

Dashboard
→ What is happening today?

Planner
→ What needs to be done?

Finance
→ Where did my money go?

Vehicle
→ What should I service next?

---

# Development Strategy

Development follows iterative sprints.

Every sprint must contain:

- Documentation
- Technical specification
- Implementation
- Review
- Commit

No feature should be implemented before being documented.

---

# Artificial Intelligence Usage

AI is used as a development assistant.

AI must never invent architecture decisions.

AI must always follow project documentation.

Whenever documentation conflicts with generated code,
documentation has higher priority.

---

# Code Quality

The project prioritizes:

Readability > Clever code

Maintainability > Short code

Consistency > Personal preferences

Simple solutions are preferred over complex abstractions.

---

# Long-Term Goal

AdamLife should eventually become the primary daily application used by its owner for:

- task planning
- project management
- finance tracking
- vehicle maintenance
- monthly statistics
- personal productivity

Every new feature must move the project closer to this goal.
